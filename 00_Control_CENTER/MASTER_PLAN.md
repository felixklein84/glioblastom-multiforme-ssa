# Felix Personal Engineering System – Master Plan

## Purpose

This project is the central operating system for upgrading my technical profile, GitHub repositories, project documentation, portfolio website, CV, and Claude Code workflows.

The goal is to transform existing projects from rough academic, student, or work-in-progress material into polished, professional showcase projects suitable for:
- job applications
- technical interviews
- master applications
- networking
- personal branding
- GitHub portfolio presentation

## Core Principle

Do not present everything I have done.

Curate the strongest projects into a coherent technical profile.

The profile should communicate:

> I am a mathematically trained technical generalist who can model complex systems, build simulations, work with data, understand real-world business constraints, and present technical work with strong visual quality.

## Quality Bar

Everything public should look professional.

Avoid:
- unfinished student-project appearance
- messy READMEs
- unclear repo names
- private or sensitive data
- overclaiming
- buzzword-heavy writing
- generic AI style
- untested code changes
- huge unreviewed refactors

Prefer:
- clear project positioning
- strong first visual impression
- honest technical depth
- clean folder structures
- reproducibility
- small commits
- good screenshots
- precise writing
- practical business or research relevance

## Workflow Per Repository

Each repository goes through this pipeline:

1. Import / clone repository
2. Create or update CLAUDE.md
3. Run repository audit
4. Create improvement roadmap
5. Create branch for upgrade
6. Implement small improvements
7. Run tests / checks
8. Review diff
9. Improve documentation
10. Add visual assets
11. Check for secrets / private data
12. Publish or merge
13. Add to portfolio website
