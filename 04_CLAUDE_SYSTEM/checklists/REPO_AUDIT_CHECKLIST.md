# REPO_AUDIT_CHECKLIST.md

## Basic Repository Information

- [ ] Repository name is understandable
- [ ] Project purpose is clear
- [ ] Main language is identifiable
- [ ] Main entry point is identifiable
- [ ] Project status is known
- [ ] Public/private intention is known

## Structure

- [ ] README.md exists
- [ ] Source code is organized
- [ ] Tests folder exists or absence is documented
- [ ] Examples exist or can be added
- [ ] Assets are separated
- [ ] Data files are separated
- [ ] Notebooks are separated from reusable code
- [ ] Generated files are not cluttering the repo

## Dependencies

- [ ] requirements.txt, pyproject.toml, package.json, or equivalent exists
- [ ] Python/Node/tool version is documented
- [ ] Dependencies are not excessive
- [ ] Installation path is clear

## Reproducibility

- [ ] Run command is documented
- [ ] Example command exists
- [ ] Test command exists or manual validation is documented
- [ ] Example output is available or planned
- [ ] External data requirements are documented

## Privacy

- [ ] No API keys
- [ ] No tokens
- [ ] No credentials
- [ ] No private emails
- [ ] No phone numbers
- [ ] No company-private data
- [ ] No raw SAP/company exports
- [ ] No confidential research data
- [ ] No local file paths
- [ ] No private images or PDFs
