# PRIVACY_RELEASE_CHECKLIST.md

Use this checklist before making any repository public, pinning it on GitHub, or linking it from the portfolio website.

## Credentials and Secrets

- [ ] No API keys
- [ ] No tokens
- [ ] No passwords
- [ ] No private keys
- [ ] No .env files
- [ ] No config files with credentials

## Personal Information

- [ ] No private email addresses
- [ ] No phone numbers
- [ ] No home addresses
- [ ] No private notes
- [ ] No personal IDs
- [ ] No unnecessary metadata in PDFs/images

## Company / Work Information

- [ ] No company-private data
- [ ] No raw SAP exports
- [ ] No supplier lists
- [ ] No customer names
- [ ] No internal process documents
- [ ] No internal URLs
- [ ] No confidential screenshots

## University / Research Information

- [ ] No confidential research data
- [ ] No private medical or biological data
- [ ] No unpublished collaborator material
- [ ] No restricted thesis material
- [ ] No private PDFs from collaborators

## Git History Risk

- [ ] Sensitive files were not committed previously
- [ ] If sensitive files were committed, history cleanup is considered
- [ ] Public release decision accounts for Git history

## Final Release Decision

- [ ] Safe to make public
- [ ] Public after minor cleanup
- [ ] Keep private
- [ ] Split into public demo repo and private original repo
- [ ] Archive only
