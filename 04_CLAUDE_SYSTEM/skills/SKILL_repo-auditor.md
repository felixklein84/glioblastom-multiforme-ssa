# SKILL_repo-auditor.md

## Purpose

Use this skill when auditing a repository before improving it.

## Core Rule

Do not edit files during the audit.

First inspect, understand, and produce a structured assessment.

## Audit Areas

- repository structure
- README quality
- dependency management
- tests and validation
- example scripts
- notebooks
- data files
- assets
- Git status
- privacy risks
- large files
- public presentation
- visual potential
- project positioning
- portfolio value

## Output Format

```markdown
# Repository Audit

## 1. Current State

## 2. Strengths

## 3. Weaknesses

## 4. Privacy / Security Risks

## 5. Reproducibility Risks

## 6. Documentation Quality

## 7. Visual Potential

## 8. Public Positioning

## 9. Upgrade Level

Choose:
- Archive Only
- Basic Public Repo
- Showcase Project
- Flagship Project

## 10. Quick Wins

## 11. Larger Improvements

## 12. Recommended First Phase
```
